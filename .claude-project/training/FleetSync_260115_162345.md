**12. Main Feature Module Flow:**

### Module: Delivery Assignment
1. Dispatcher views unassigned deliveries on the board
2. System suggests optimal driver based on location and capacity
3. Dispatcher drags delivery to driver column (or accepts suggestion)
4. System calculates ETA and optimizes route
5. Driver receives push notification with new assignment
6. Driver accepts or requests reassignment
7. Status updates to "Assigned" on all views

### Module: Proof of Delivery
1. Driver arrives at destination, taps "Arrived"
2. System logs GPS coordinates and timestamp
3. Driver completes delivery, taps "Complete Delivery"
4. Camera opens for photo capture (package at door)
5. Signature pad opens for recipient signature (optional based on delivery type)
6. Driver adds notes if needed
7. System uploads proof and updates delivery status
8. Customer receives delivery confirmation notification

### Module: Vehicle Issue Reporting
1. Driver notices vehicle issue during route
2. Taps "Report Issue" button
3. Selects issue category (engine, tire, brakes, etc.)
4. Takes photo of issue
5. Adds description and severity level
6. System notifies Fleet Manager immediately
7. Fleet Manager reviews and creates maintenance ticket
8. If critical, dispatcher reassigns remaining deliveries

### Module: Route Optimization
1. Dispatcher selects multiple deliveries for a driver
2. Taps "Optimize Route"
3. System considers: traffic, time windows, vehicle capacity, driver hours remaining
4. Presents optimized route with estimated time savings
5. Dispatcher approves or manually adjusts
6. Route pushed to driver app with turn-by-turn ready
