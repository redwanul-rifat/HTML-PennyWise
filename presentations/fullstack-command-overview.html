Fullstack Pipeline Orchestrator

Team Training Guide

Potential Inc • January 2026

What is /fullstack?

  • A skill-chain orchestrator
  • Runs the full development lifecycle
  • Tracks progress in PIPELINE_STATUS.md
  • Delegates to specialized skills per phase
  • One command to go from init to ship

The 9 Pipeline Phases

init → prd → database → backend → frontend → integrate → test → qa → ship
  • init - Project setup (.claude-project/, .claude/)
  • prd - Convert PRD to PROJECT_KNOWLEDGE.md
  • database - Design entities and migrations
  • backend - Implement API endpoints
  • frontend - Build React screens
  • integrate - Connect frontend to backend
  • test - Generate E2E tests
  • qa - Run design QA (95% pass required)
  • ship - Deploy to production

Phase Flow & Prerequisites

  • init has no prerequisites
  • prd requires init
  • database requires prd
  • backend requires database
  • frontend requires prd (parallel with backend!)
  • integrate requires backend + frontend
  • test requires integrate
  • qa requires test
  • ship requires qa

Quick Start Commands

/fullstack my-project

Show current pipeline status

/fullstack my-project --run

Run the next pending phase

/fullstack my-project --phase backend

Run a specific phase

/fullstack my-project --run-all

Run all remaining phases automatically

/fullstack my-project --reset database

Reset a phase to pending status

Tech Stack Configuration

  • Configured in PIPELINE_STATUS.md
  • Backend: nestjs or django
  • Frontends: react, react-native
  • Skill paths resolved automatically
tech_stack:
  backend: nestjs
  frontends: [react-native, react]
  dashboards: [admin]

Frontend: Three Paths

  • Design from scratch
    • Generate design prompts from PRD
    • Create designs externally (Aura, v0, Gemini)
  • Convert from Figma
    • Use MCP tools to read Figma designs
    • Auto-generate React components
  • Convert from HTML
    • Convert existing HTML/Tailwind templates
    • Transform to React components

Agent asks which path to use

Status Tracking

  • PIPELINE_STATUS.md tracks everything
  • Status icons:
    • Pending - Not started
    • In Progress - Currently running
    • Complete - Successfully finished
    • Failed - Needs attention
  • Output column shows deliverables
  • Execution log records all runs
  • Never lose your progress

Ralph Integration

  • Some phases delegate to Ralph
  • Item-level iteration (48 screens, 29 tests)
  • Ralph runs until completion criteria met
  • QA phase needs 95% pass rate
/ralph e2e-tests my-project --incremental
/ralph design-qa my-project --incremental

Example Workflow

# Start a new project
/fullstack my-app --run   # init
/fullstack my-app --run   # prd
/fullstack my-app --run   # database
/fullstack my-app --run   # backend
/fullstack my-app --run   # frontend
/fullstack my-app --run   # integrate
/fullstack my-app --run   # test
/fullstack my-app --run   # qa
/fullstack my-app --run   # ship

# Or just run everything:
/fullstack my-app --run-all

Key Takeaways

  • One command orchestrates everything
  • 9 phases from init to ship
  • Tracks progress automatically
  • Supports multiple tech stacks
  • Delegates to Ralph for iteration
  • Just run /fullstack and follow along!
